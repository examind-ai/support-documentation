# Build

Building an assessment in EXAMIND follows a simple flow:

1. **Create questions** in the question builder — choosing from eight question types, and optionally making them *dynamic* so every student gets a unique variant.
2. **Organize and reuse them** in the question bank — share questions across courses and pool them to randomize what each student sees.
3. **Assemble them into an assessment** — an exam, quiz, practice set, essay, or simulation.

The dynamic question engine runs through all of this: it's what lets a single question become a unique variant per student, which is core to EXAMIND's approach to academic integrity.

## In this section

* [**Question Bank**](question-bank/README.md) — import, share, and pool your questions for reuse across courses.
* [**Question Builder**](question-builder/README.md) — create questions across the eight question types, including dynamic, per-student variants.
* [**Assessments**](assessments/README.md) — assemble questions into exams, quizzes, practice, essays, and simulations.
